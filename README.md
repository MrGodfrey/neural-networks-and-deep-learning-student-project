## 可以贡献的内容

欢迎通过 Pull Request 贡献课程相关内容，包括但不限于：

1. 自己制作的 slides
2. 有趣的编程项目，尤其是与机器学习相关的项目
3. 可用于实验课、课堂演示或自学的 Jupyter Notebook
4. 其他高质量、与课程相关的学习材料

## Pull Request 贡献指南

请使用 Pull Request 的方式提交内容。

1. Fork 本仓库到你自己的 GitHub 账号。
2. 在你的 fork 中创建一个命名清晰的新分支。
3. 添加你的材料，并在必要时补充简短说明，帮助我理解内容的用途和背景。
4. 提交你的修改并推送到你的 fork。
5. 向本仓库发起 Pull Request，并在描述中说明你提交的内容、适用场景，以及任何需要我特别注意的事项。
6. 我会审核 Pull Request，并决定是否合并到课程项目仓库中。

## 重要说明

一旦通过 Pull Request 或其他方式向本仓库提交内容，即视为同意下方的版权转让政策。也就是说，提交 Pull Request 后，版权将自动转让给课程任课教师。

## 版权转让政策

向本仓库提交项目或材料时，包括但不限于通过 Pull Request 或直接上传，即表示你同意以下条款：

1. 你在此将你的提交内容的全部著作权及相关权利转让给本课程的任课教师。
2. 任课教师有权无需另行获得你的许可，对所提交的作品进行使用、修改、分发、再许可和商业化。
3. 你确认你的提交内容为原创作品。如果你包含第三方代码，你必须确保其与本政策兼容，并已进行适当署名。
4. 如果你不同意这些条款，请不要向本仓库提交项目或材料。课程可另行提供其他提交方式。

本政策自提交之时起生效，并对提交行为具有约束力。
